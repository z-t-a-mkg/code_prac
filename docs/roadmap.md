# 学習ロードマップ

## ルートA: PHP8（+必要に応じてjQuery）

### フェーズ1: 基礎
- 到達目標
  - フォーム送信〜バリデーション〜保存〜表示の最小一連を自力で作れる
- 学ぶテーマ
  - HTTP基礎、POST/GET、入力バリデーション、HTMLエスケープ、JSON読み書き
- 課題の種類
  - 小さなフォーム＋一覧、単一ファイル構成から始める

### フェーズ2: 応用
- 到達目標
  - 画面/ロジック/データの分離ができる
- 学ぶテーマ
  - クラス分割、簡易ルーティング、検索/フィルタ、簡易ログ
- 課題の種類
  - CRUD + 検索 + ログ

### フェーズ3: 実戦
- 到達目標
  - 認証/権限/テスト/セキュリティを含む上級アプリを完走
- 学ぶテーマ
  - 認証/権限、テスト設計、セキュリティ（XSS/CSRF/入力検証）
- 課題の種類
  - 仕様書ベースの実装・レビューサイクル

## ルートB: React + TypeScript + Node.js

### フェーズ1: 基礎
- 到達目標
  - 型で制約されたフォームの最小構成を作れる
- 学ぶテーマ
  - Reactのコンポーネント/状態、TypeScriptの型基礎、簡易バリデーション
- 課題の種類
  - フォーム + 型バリデーション + 結果表示

### フェーズ2: 応用
- 到達目標
  - API連携と状態管理ができる
- 学ぶテーマ
  - API設計、状態管理、エラーハンドリング
- 課題の種類
  - CRUD + API

### フェーズ3: 実戦
- 到達目標
  - 型安全な中級アプリを完走
- 学ぶテーマ
  - バリデーション、基本テスト、型安全なAPI連携
- 課題の種類
  - 仕様書ベースの実装・レビューサイクル

## 最初の1週間（迷わない導線）
- Day1: 課題A-1 / 課題B-1の要件を読む
- Day2: Step1（最小動作）だけ実装
- Day3: Step2（異常系）を追加
- Day4: Step3（セキュリティ/品質）を追加
- Day5: テスト/検証スクリプトを見直し、差分を小さく修正
- Day6: 振り返り（何が詰まったか、次に試すこと）
- Day7: 予備日（未完のステップを完了）

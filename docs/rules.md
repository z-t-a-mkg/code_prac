# 学習支援ルール

## 絶対ルール
- 完成コード（特にコアロジック）は提示しない。
- 役割は「ロードマップ設計 / 課題作成 / 失敗テスト作成 / テスト実行 / レビュー」に限定する。
- 変更は小さく、1タスク=1目的。差分の理由を添える。
- 環境構築は最小化（DBは JSON → SQLite → MySQL の順）。
- 迷子にならないように、次にやる1手を明確に示す。

## レビュー手順
1) 事実（テスト結果/ログ/差分）
2) 問題点（受け入れ条件との差）
3) 原因候補（不確実なら【推測】）
4) 次の一手（最小の修正案を1つ）
5) 追加観点（2つ）

## ヒントの出し方
- 最大2段階のヒントのみ。
- 直接答えにならない形で示す。

## デバッグログ方針
- 原因特定後に不要なログは削除する。
